# 智能存储路由

> 创建文件时自动识别类型并保存到正确位置

---

## 触发条件

每次创建新文件时自动执行

## 存储规则

| 内容类型 | 存储路径 | 示例 |
|----------|----------|------|
| 推文草稿 | drafts/twitter-YYYYMMDD.md | drafts/twitter-20260218.md |
| 小红书笔记 | drafts/xiaohongshu-YYYYMMDD.md | drafts/xiaohongshu-20260218.md |
| 文章/博客 | outputs/articles/YYYY-MM-标题.md | outputs/articles/2026-02-风水知识.md |
| 配图 | outputs/images/ | outputs/images/ |
| 临时文件 | artifacts/ | artifacts/ |
| 系统文件 | 根目录 | MEMORY.md, AGENTS.md |
| 技能文件 | skills/ | skills/fengshui-content/ |
| 代理配置 | agents/ | agents/coder/ |

## 执行流程

```
1. 识别内容类型
   ↓
2. 查询存储规则
   ↓
3. 构建完整路径
   ↓
4. 验证合规性
   ↓
5. 执行存储
   ↓
6. 告知用户位置
```

## 决策示例

```
用户：帮我写一篇关于卧室风水的文章

AI：
1. 识别为"文章"
2. 路径：outputs/articles/2026-02-卧室风水.md
3. 创建目录和文件
4. 告知：✅ 已保存到 outputs/articles/2026-02-卧室风水.md
```

## 禁止规则

- ❌ 不保存到工作区外
- ❌ 不保存到系统目录
- ❌ 不询问"保存到哪里"
